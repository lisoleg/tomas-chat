### AEGIS 四阶段流水线（简化版）

```python
# extend_hypergraph.py（升级后）
class AEGISEngine:
    def evolve(self, trajectory, current_harness):
        # 1. Digester
        failed_edges = self.digester(trajectory)
        # 2. Planner
        proposals = self.planner(failed_edges, current_harness)
        # 3. Evolver
        candidate = self.evolver(current_harness, proposals)
        # 4. Critic+Gate
        accept, reason = self.critic_gate(candidate, trajectory)
        return (candidate, reason) if accept else (None, reason)
```

## 数据库架构评估（2026-06-20 凌晨）

### 用户问题
是否应该采用图数据库（Neo4j）或自研超图数据库，支持拟群/拟阵遍历查询？

### 参考材料（3篇论文 + 1个代码库）
1. **本征时空数据库**（净光哥，公众号）— 34页，核心：时空是数据本体形式非属性；四元数几何化；比 PostgreSQL 快15.8x
2. **ChainDB: 面向AGI的关系索引区块链数据库**（章锋，28页，含代码）— "信息在关联中，不在实体中"；RelationIndex + Ftel度量；5层架构；94/94测试通过；GitHub: lisoleg/chain-db
3. **论AI原生存算通讯一体化数据库**（章锋，12页）— 波粒二象性、阿卡西记录、虹光身架构（图片PDF，内容提取有限）
4. **TOMAS 现有模块** — `eml_dimred/hyperedge.py`(HypEdge n-ary), `matroid.py`(κ-Gate贪心剪枝), `gpct.py`, `itc.py`, `brown_miklos.py`

### 分析结论
**不推荐换 Neo4j**（无超边/拟阵原生支持，与 EML 格式不兼容）
**推荐 B+C 融合路线**：
- **B**: 基于 ChainDB RelationIndex 扩展为 TOMAS-HypergraphDB（HypEdge 持久化 + Matroid 引擎）
- **C**: 分层混合 — L1 SQLite 冷数据(101M行) + L2 内存超图索引(热数据子集~10K) + L3 Matroid 推理
