## 死零/MUS/κ-Snap 机制实现 ✅

基于章锋《开源大模型军备竞赛下的 TOMAS 战略》（2026-06-15）微信公众号文章，实现 TOMAS 核心 IP：

### 新增模块：`tomas_agi/sim/dead_zero_mus.py`（~600 行）

| 类 | 功能 | 对应文章章节 |
|------|------|-------------|
| `DeadZeroChecker` | 死零校验：ℐ(e) < θ_dead ⇒ [DEAD_ZERO_REJECT] | 第三章 EML 注入机制 |
| `MUSArbitrator` | MUS 仲裁：悖论对双存 ⇒ [MUS_ACTIVE] | 第三章 + 第五章 |
| `KSnapDecider` | κ-Snap 决策：优先最高 ℐ(e)，平局时 MUS ⇒ 保留延续性 | 第三章 EML 注入机制 |
| `DeadZeroMUSGate` | 统一门控器（集成上述三大机制） | 核心 IP |

### 集成到 `token_bridge.py`

- `InferenceEngine.__init__()`: 新增死零/MUS 参数
- `InferenceEngine._init_dead_zero_mus_gate()`: 初始化门控器
- `InferenceEngine._apply_dead_zero_mus_gate()`: 应用门控检查
- `InferenceEngine._extract_matched_edges()`: 从查询结果提取边
- `InferenceEngine.generate_response()`: 在路由前加入死零/MUS 检查
- CLI 参数：
  - `--disable-dead-zero`: 禁用死零校验
  - `--theta-dead`: 设置死零阈值（默认 0.15）
  - `--disable-mus`: 禁用 MUS 仲裁
  - `--disable-k-snap`: 禁用 κ-Snap 决策
