## κ-Gate 语义剪枝实现 ✅
- **理论基础**: HyperGNN 文章 + 用户论文论证 → I(X) 信息存在度 + κ-Gate 剪枝
- **models.py**: KnowledgeTriple 添加 `i_weight` 列（Float, DEFAULT 1.0, NOT NULL）
  - 新增 `idx_triples_i_weight` 索引
  - 修复 `server_default="1.0"` bug（此前 INSERT OR IGNORE 因 NOT NULL 无 DEFAULT 导致 0 行插入）
- **import_ownthink_sqlite.py**: `compute_i_weight()` 后计算函数
  - 启发式：`i_weight = 1.0 + ln(1 + subject_freq) / 10.0`，范围 [1.0, ~3.0]
  - 两步：CREATE TABLE _subject_freq → JOIN UPDATE i_weight → DROP TABLE
  - 分批 UPDATE（5M 行/批）+ 进度打印 + 分布抽样验证
  - `--skip-i-weight` 跳过计算；修复 CSV 表头跳过逻辑
- **server.py API 扩展**:
  - `/api/knowledge/triples?min_i_weight=N` — κ-Gate 过滤，按 i_weight DESC 排序
  - `/api/knowledge/graph?min_i_weight=N&subject=X` — 过滤 + 子图展开 + i_weight 返回
- **token_bridge.py BFS 剪枝**:
  - `extract_subgraph(vertex_ids, radius, kappa)` — 仅遍历 I(X) ≥ κ 的顶点/边
  - `query()` / `generate_response()` 透传 kappa
  - `--kappa` CLI 参数（默认 0.0 = 不剪枝）
