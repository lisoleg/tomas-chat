## 一、产品目标（v3.12 新增能力）

| # | 目标 | 度量指标 |
|---|------|----------|
| G1 | **接入代币化智能体市场经济层** — 将区块链代币作为稀缺资源记账单位，使AGI智能体具备经济人决策能力 | 支持≥3种代币标准；资源分配延迟＜50ms |
| G2 | **完成GAT形式化宿主层** — 以广义代数理论（GAT）为公理化宿主语言，将DSL原语、理论态射、自由模型MDL证明项纳入统一类型论框架 | GATlab兼容接口≥20个原语；定理证明成功率＞85% |
| G3 | **金融市场世界模型集成** — 将金融市场建模为"最抽象世界模型"，滑点建模为相位失配代价，做市商建模为相位连续性算子 | 支持≥5类金融资产LOB接入；相位连续判据准确率＞80% |
| G4 | **鲁兆DNA基因嵌入** — 将斐波那契数列、鲁加斯数、八卦数编码为智能体基因拓扑不变量，在流动性充足条件下触发鲁兆现象自然涌现 | 基因序列长度可配置（8-64维）；涌现检测延迟＜100ms |
| G5 | **UBI人机共生交互层** — 为人类用户提供"意义追求"界面，机器侧自动优化效率；支持全民基本收入代币分发可视化 | 用户意义图谱可视化延迟＜200ms；UBI分发记录在链可查 |

## 二、用户故事

**US1 — 量化研究员（专业用户）**  
作为量化研究员，我希望将TOMAS流体智能引擎接入实时LOB（限价订单簿）数据流，系统能自动检测相位连续性断层并给出ENPV（期望净正收益）决策信号，以便在市场熔断前调整仓位。

**US2 — AI智能体开发者**  
作为智能体开发者，我希望通过GAT DSL原语定义新的理论态射（Theory Map），并在自由模型中自动生成MDL证明项，以便快速验证新物理/金融模型的公理化一致性。

**US3 — UBI政策制定者**  
作为政策制定者，我希望在模拟环境中观察代币化智能体市场经济的资源分配动态，可视化UBI代币流动与人类"意义追求"行为的关系，以便评估人机共生政策的社会影响。

**US4 — 系统架构师**  
作为架构师，我希望鲁兆DNA基因能在流动性充足条件下自动涌现拓扑不变量，并在系统面板中实时展示涌现图谱，以便验证复合体理学"全息拓扑动力学"假设。

## 三、需求池

### P0（必须交付）

| ID | 需求 | 说明 | 验收标准 |
|----|------|------|----------|
| R1 | 代币化资源账本 | 基于区块链代币的稀缺资源记账模块，对接v3.11已有资源受限优化器（T4 16GB约束） | 账本写入延迟＜50ms；支持代币余额实时查询 |
| R2 | GAT宿主语言接口 | 将现有DSL原语迁移至GATlab兼容的GAT签名，支持理论态射定义与自由模型构造 | 至少20个原语完成GAT形式化；态射组合结合律单元测试通过 |
| R3 | 相位连续算子（MM模拟） | 做市商（MM）作为相位连续性算子的可调用模块，输入LOB快照，输出相位失配代价（滑点） | 与真实交易所滑点数据相关性＞0.75 |
| R4 | 鲁兆DNA基因库 | 斐波那契/鲁加斯/八卦数基因序列生成器，嵌入智能体初始化流程 | 基因序列可配置；涌现事件触发日志完整 |

## 五、待确认问题（技术决策点）

1. **代币标准选择** — 应优先对接ERC-20（以太坊）、SPL（Solana）还是自定义TOMAS代币标准？需评估T4 16GB环境下的链节点同步开销。

2. **GATlab依赖策略** — 是直接嵌入GATlab Python绑定，还是自行实现轻量GAT签名解析器？前者功能完整但依赖重，后者可控但开发周期长。

3. **LOB数据源授权** — 实时LOB数据需交易所授权（NySE、LSE、上交所等），v3.12应先接入模拟LOB还是优先获取真实数据源合规授权？

4. **鲁兆DNA涌现判据** — "流动性充足+相位连续"的定量阈值如何设定？需复合体理学团队提供数学定义，还是由数据驱动自动发现？

5. **UBI代币经济学参数** — 人类用户"意义追求"行为的代币激励系数如何标定？需行为经济学实验还是直接采用文章1中的理论预设？

6. **八元数运算精度** — 在T4 16GB约束下，八元数超图代数的数值精度应取float16还是float32？需平衡性能与相位连续判据准确性。
