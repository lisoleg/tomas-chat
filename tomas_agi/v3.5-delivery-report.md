## 三、关键修复

### 3.1 EMLSemZip 测试修复
**问题**：`tests/test_new_modules.py` 中 `TestEMLSemZip` 测试调用错误的 API  
**原因**：
- `compress()` 方法需要 `kb: EMLLiteKB` 参数（测试未传入）
- 不存在 `get_state()` 方法（错误地调用）

**修复**：
- 添加 `self.kb = EMLLiteKB()` 初始化
- 修正 `test_compress` 调用 `compress(self.hg, self.kb)`
- 替换 `test_get_state` 为 `test_compress_and_decompress`（验证压缩后解压恢复）

**状态**：✅ 2/2 测试通过

---

### 3.2 compute_i_weight.py 业务逻辑修复
**问题 1**：`WHERE i_weight = 1.0` 浮点比较失败  
**原因**：IEEE 754 浮点表示误差导致 `1.0` 不匹配  
**修复**：使用容差比较 `WHERE ABS(i_weight - 1.0) < 0.0001`

**问题 2**：UPDATE 语句未过滤已计算行  
**原因**：`UPDATE knowledge_triples SET i_weight = ? WHERE subject = ?` 会更新该 subject 的所有行（包括已计算权重的行）  
**修复**：添加 `i_weight` 条件：
```sql
UPDATE knowledge_triples SET i_weight = ? 
WHERE subject = ? AND ABS(i_weight - 1.0) < 0.0001
```

**问题 3**：循环结束后未处理最后一个 subject  
**原因**：`for row in cur:` 循环结束后，`current_subject` 指向最后一个 subject，但未添加到 `batch`  
**修复**：在循环结束后添加 "处理最后一个 subject" 逻辑

**问题 4**：事务管理错误  
**原因**：`isolation_level = None` 模式下 SELECT 不在事务中，错误使用 `ROLLBACK`  
**修复**：移除多余的 `ROLLBACK`，在需要时使用 `conn.close()`

**状态**：✅ 5/5 测试通过
